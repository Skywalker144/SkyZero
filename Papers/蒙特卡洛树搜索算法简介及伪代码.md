# 蒙特卡洛树搜索

​    **蒙特卡洛树搜索**（英语：Monte Carlo tree search；简称：**MCTS**）是一种用于某些决策过程的**启发式搜索算法**，最引人注目的是在游戏中的使用。

## MCTS算法流程

​    MCTS在走一步棋的时候会进行上千次模拟，首先将当前棋局作为根节点，每次模拟都会进行 **选择**、**扩展**、**仿真**、**反向传播** 四个步骤：

```python
# 将当前棋局作为根节点。to_play代表轮到谁走子。1代表黑方(先手)，-1代表白方(后手)
root = Node(
	state,
	to_play
)
for _ in range(num_simulations):
	1. 选择
	2. 扩展
	3. 仿真(模拟)
	4. 反向传播
```

​    接下来具体介绍每个步骤的内容。

***

### 1、选择

​    从根节点开始，选择**UCT**值最大的子节点，直到选择到一个**未被完全扩展的**节点或者终局状态节点

```python
# 1、选择
while not node.state.is_terminal() and node.is_fully_expanded():
	node = select(node)

def select(node):
	max_uct = -inf
	best_child = None
	for child in node.children:
		if get_uct(child) > max_uct:
			max_uct = get_uct(child)
			best_child = child
	return best_child

def get_uct(node, c=1.414):
	q = -node.v / node.n
	# 对于父节点来说，当前节点的收益需要取反；因为子节点的胜利即是父节点的失败
	u = c * sqrt(
		ln(node.parent.n) / node.n
	)
	
	q = (q + 1) / 2  # UCT的前提是节点收益在[0, 1]之间，估在此归一化
	return q + u
```

**UCT** (Upper Confidence Bound applied to Trees) 公式如下：
$$
UCT(i) = \frac{v}{n_i} + C \cdot \sqrt{\frac{ln(N)}{n_i}}
$$
$v$：累计价值

(胜利$v+1$，失败$v-1$；该价值是**相对于当前节点玩家**来说的价值。比如最终黑方胜利，当前节点轮到白方下棋，那么对于当前节点（白方）来说就是失败)

$n_i$：节点i的访问次数

$C$：探索系数，常设为$\sqrt{2}$

$N$：父节点的访问次数

> UCT公式平衡了利用与探索：
>
> ​    第一项 ${v}/{n_i}$ 是平均价值，引导算法趋向于**已知的最优路径**；
>
> ​    第二项是探索项，引导算法趋向于**访问次数较少的路径**，以防止错过那些虽然现在看起来一般、但可能隐藏着更优解的路径。（节点 $i$ 访问次数少，分母就小，整体就大）
>
> ​    例如，一个搜索过程可能会出现如下变化：
>
> 1、**初期：** 如果一个新路径被发现且胜率不错，**利用项**占据主导，算法会连续尝试该路径。
>
> 2、**中期：** 随着该路径被访问次数（$n_i$）增加，**探索项**的分母变大导致数值减小。与此同时，由于父节点总数 $N$ 还在增加，那些长期没被访问过的节点的**探索项**会逐渐变大。
>
> 3、**转折：** 当某个未充分探索路径的“潜力值”超过了当前最优路径的高胜率值时，算法就会转而去尝试那个未知的节点。
>
> ​    这种机制保证了MCTS既能深入挖掘有潜力的路径，又不会完全漏掉其他可能的奇招。





### 2、扩展

​    如果当前节点为非终局节点(即未完全展开的节点)，则随机选择一个动作，扩展出执行这个动作之后的节点

```python
# 2、扩展
if not node.state.is_terminal():
	node = expand(node)

def expand(node):
	action = node.state.get_random_untried_action()  # 随机选择一个尚未扩展过的动作
	to_play = node.to_play
	child_node = Node(
		state=node.state.get_next_state(action, to_play),
		to_play=-to_play,
		action_taken=action,
		parent=node
	)
	node.children.append(child_node)
	return child_node
```



### 3、模拟

​    将该节点按照随机策略玩到终局，获得最终赢家

```python
# 3、模拟
winner = simulate(node.state, node.to_play)  # 绝对视角下的winner
value = winner * node.to_play  # 转换成对于当前节点的相对胜负

def simulate(state, to_play):
	while not state.is_terminal():
		action = state.get_random_action()
		state = state.get_next_state(action, to_play)
		to_play *= -1
	winner = state.get_winner()
	return winner
```



### 4、反向传播

​    将赢家按照相对视角获得各节点的价值，一路往上从当前节点传到根节点。

```python
 # 4、反向传播
backpropagate(node, value)

def backpropagate(node, value):
	while node is not None:
		node.v += value
		node.n += 1
		value *= -1  # 转换视角，因为当前节点和其父节点和to_play相反，所以相对价值也相反
		node = node.parent
```



## 最终策略

​    选取根节点的访问次数最高的子节点，该子节点的action_taken即为最终动作。

```python
def search(state, to_play):
	root = Node(state, to_play)

	for _ in range(num_simulations):
		node = root

		# 1、选择
		while not node.state.is_terminal() and node.is_fully_expanded():
			node = select(node)

		# 2、扩展
		if not node.state.is_terminal():
			node = expand(node)

		# 3、模拟
		winner = simulate(node.state, node.to_play)  # 绝对视角下的winner
		value = winner * node.to_play  # 转换成对于当前节点的相对胜负

		# 4、反向传播
		backpropagate(node, value)

	best_child = max(root.children, key=lambda c: c.n)
	return best_child.action_taken
```



### 注意

1. UCT选择时，Q值需要取反，因为父节点视角下的收益与子节点视角相反：子节点赢了，意味着父节点输了。
2. 收益范围假设：UCT算法的前提是节点收益在$[0, 1]$之间
